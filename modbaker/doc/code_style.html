Code Style

No CamelCase

Function names should not be coded in CamelCase.
Please use underscores instead.

Example: do_something();

Class names

Class names start with a c_

Example: class c_class {};

Member variables

Member variables (variables within a class definition) start with a m_

Example: 

	class c_test()
	{
		private:
			int m_test_integer;
	};

Text indent

Please use tabs to indent text, not spaces.

{ and }

To make the code more readable, place the brackets { and } in an own line.

Example: 

	int one()
	{
		return 1;
	}

For and If: ( and )

Do not use spacing within the brackets

Example: 

	if(test == 1)
	{
		do_domething();
	}
